<analysis>**original_problem_statement:** 
The user wants to build a full-stack auditing system for the food industry called Infinit-Audit.

**PRODUCT REQUIREMENTS:**
- **User Roles & Authentication:**
    - Three roles: Admin, Audit Creator, Normal User (view reports/run audits).
    - JWT-based authentication.
    - Admins can create companies and assign users to them.
- **Data Isolation:** Users can only see data (audits, response sets) created by users within their own company.
- **Core Features:**
    - **Groups:** Create reusable response sets (e.g., Pass/Fail, Yes/No) and audit type categories. Allow flagging responses as negative.
    - **Create Audit:** Build audit templates with a name, description, and questions. Questions can use predefined response groups.
    - **Run Audit:** Execute an audit based on a template. Must support photo uploads. Comments are mandatory for negative or fail responses.
    - **Reports:** View a list of completed audits. Click to open a detailed view showing questions, responses, comments, and scores.
- **Enhancements & Advanced Features:**
    - **Timestamps:** All dates and times must be recorded in the UK timezone.
    - **Scoring:** Audits should have an optional scoring mechanism.
    - **PDF Export:** Ability to download completed audit reports as PDFs.
    - **Scheduling:** Admins/Creators can schedule audits to be completed in the future, with email reminders.
    - **Bulk User Management:** Admins can bulk import users via a CSV file.
    - **Company Dashboard:** A dashboard for admins to view company-wide compliance trends and analytics.
    - **Offline Capability:** The application should ideally have offline functionality.
- **Design:**
    - A clean, modern, minimalistic look.
    - Both light and dark themes.

**User's preferred language**: English
